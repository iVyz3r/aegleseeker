# Contributing to Aegleseeker Client

Thank you for your interest in contributing! Your help is appreciated and will improve the project for everyone.

## How to Contribute

1. **Fork the repository** and clone it locally.
2. **Create a new branch** for your changes:

```bash
git checkout -b feature/your-feature-name
```

3. **Make your changes** following the existing code style.
4. **Test your changes** to make sure everything works.
5. **Commit your changes** with clear messages:

```bash
git commit -m "Add explanation for alloc operations"
```

6. **Push your branch**:

```bash
git push origin feature/your-feature-name
```

7. **Open a Pull Request** on GitHub, describing your changes clearly.

## Code Guidelines

* Keep functions modular and readable.
* Use clear variable names.
* Document complex operations, especially memory manipulations like `alloc`, `jmp`, or pointer arithmetic.
* Avoid changing formatting unnecessarily.

## Reporting Issues

* Use GitHub Issues for bugs or suggestions.
* Provide steps to reproduce bugs whenever possible.
* Include screenshots, logs, or examples if relevant.
